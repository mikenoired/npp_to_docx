# npp_to_docx

## Архитектура

Проект теперь разделён на логические слои:

- `src/core` — общий движок обработки SVG/DB/DOCX
- `src/electron` — main/preload слой Electron, доступ к файловой системе и IPC
- `src/web/client` — React 19 renderer внутри Electron
- `src/shared` — общие типы IPC/API между main и renderer

`core` теперь запускается прямо внутри Electron-приложения. В production-цепочке больше нет отдельного Bun HTTP server и отдельного web-бинарника.

## Установка

Для разработки и сборки требуется установить зависимости проекта. После упаковки конечный пользователь получает самостоятельное desktop-приложение без внешнего сервера.

```bash
npm install
```

Можно использовать и другой package manager, если он совместим с `package.json`.

## Запуск

```bash
npm run dev
```

Это поднимает:

- Vite renderer на `http://127.0.0.1:5173`
- Electron desktop shell
- встроенный `core` в main process

HTTP API и внешний браузер больше не используются.

## Использование

После запуска приложения:

1. Укажите входную и выходную директории или выберите их через диалог.
2. Во входной директории должны лежать `PLS_ANA_CONF.dmp`, `PLS_BIN_CONF.dmp` и папка `svg`.
3. Нажмите `Подготовить базу данных`.
4. Нажмите `Запустить обработку`.
5. После завершения используйте вкладку поиска по подмодели.

По умолчанию приложение создает workspace в `Documents/npp_to_docx` и хранит там:

- `input/`
- `output/`
- `logs/`

## Сборка приложения

```bash
npm run dist
```

Результат:

- macOS: `.app`, `.dmg`, `.zip`
- Linux: `AppImage`, `deb`

Итоговая сборка содержит один desktop runtime на платформу, внутри которого уже находится UI, `core` и доступ к файловой системе.
